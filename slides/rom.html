Preliminary: Modeling Hash Functions
Hash functions $\approx$ truly random functions
Model function evaluation as oracle queries.
Resource consumed $\sim$ query complexity.
Example: Classical analysis of proof of work algorithm
Keep a database of oracle queries + responses.
An algorithm cannot win unless there is a response $y_i=\underline{0^ns}$ in the database.
This takes $2^n$ queries on average.
Classical random oracle model.
